# 專案結構與技術說明

## 一、檔案職責

| 檔案 | 職責 | 依賴 |
|------|------|------|
| `index.html` | 進入點：DOM 骨架（塵埃/暈影/顆粒層、標題、結局容器、彈窗）、載入 css/js | — |
| `assets/css/style.css` | 全部樣式：色彩變數、動畫、場景/熱點/HUD/對話/物品欄/謎題元件/結局/響應式 | — |
| `assets/js/data.js` | **純資料**：CHARS、TITLE、ROOMS、ENDING、HINTS、MISCONCEPTION、RANKS | — |
| `assets/js/audio.js` | Web Audio 音效合成，匯出全域 `SFX` | 呼叫時讀 `S.muted` |
| `assets/js/engine.js` | 引擎：狀態 `S`、場景渲染、對話系統、HUD、物品欄、彈窗、toast、粒子、提示、筆記 | data、audio |
| `assets/js/puzzles.js` | 四關謎題實作、熱點分派 `openHot`、拖曳/轉盤助手、過關轉場 | data、engine |
| `assets/js/main.js` | 結局手稿與稱號 `showEnding`、塵埃、開場綁定與啟動 | 全部 |

> 載入順序（見 index.html 底部）：data → audio → engine → puzzles → main。

## 二、執行流程（資料流）

```
DOMContentLoaded
   └─ boot()  (main.js)
        ├─ initDust()                漂浮塵埃
        └─ 綁定「推開研究塔大門」按鈕
             └─ 點擊 → 解鎖 AudioContext → 淡出標題 → renderRoom(1)

renderRoom(n)  (engine.js)
   ├─ 以 ROOMS[n-1].bg 設整頁背景大圖
   ├─ 生成熱點（hots）→ 綁 openHot(n,id)
   ├─ 注入 HUD + 物品欄
   └─ playDialogue(intro) → 結束後：n===3 initDarkroom()／n===4 initGate()

openHot(n,id)  (puzzles.js)  →  對應謎題函式
   解出 → gotoNextFrom(n)  →  renderRoom(n+1)  或  showEnding()
```

## 三、狀態物件 S（engine.js）

```js
const S = {
  scene,        // 目前畫面
  room,         // 目前關卡編號
  hints,        // 已用提示次數（影響結局稱號）
  inv,          // 物品欄陣列 [{icon,name,desc}]
  t0, tHandle,  // 計時器起點與 interval handle
  muted,        // 靜音
  doneRoom      // { 1:1, "1b":1, 2:1, ... } 已解狀態旗標
};
```
> 全部僅存記憶體，**不使用 localStorage / sessionStorage**（符合部署限制）。

## 四、如何新增一個關卡（擴充系列單元）

以新增「因式分解」為例：

1. **加資料**：在 `data.js` 的 `ROOMS` 加一個物件，指定 `bg`、`chap`、`formula`、`page`、`intro`、`hots`。
   若需要提示，於 `HINTS` 加對應編號的三段文字。
2. **加謎題**：在 `puzzles.js` 的 `openHot()` 分派新的熱點 id 到你的新函式，並實作該函式；
   解出後呼叫 `gotoNextFrom(該關編號)`。
3. **加素材**：把新場景圖放進 `assets/images/`，於資料的 `bg` 指到它。
4. 若是最後一關，`gotoNextFrom` 會自動導向 `showEnding()`；可在 `ENDING.cards` 增卡片。

> 引擎（engine.js）與音效（audio.js）通常**不需修改**，因為它們不含任何硬編內容。

## 五、共用元件速查

| 功能 | 呼叫 |
|------|------|
| 彈窗 | `modal(html)` / `closeModal()`（含 `[data-x]` 自動綁關閉） |
| 提示訊息 | `toast(msg, "ok"/"err")` |
| 答對回饋 | `good(msg, x, y)`（金色粒子＋上行音） |
| 答錯回饋 | `bad(node, msg)`（震動＋低鳴） |
| 取得道具 | `addItem(icon, name, desc)` |
| 拖曳拼合 | `wireDrag(tilesSel, onAllPlaced)` |
| 數字轉盤 | `buildReels(hostId, len)` / `readReels(hostId)` |
| 音效 | `SFX.click/paper/gear/unlock/err/good/win/tick()` |

## 六、瀏覽器相容

- 使用 ES6+（const/let、箭頭函式、樣板字串、Web Audio、CSS mask）。
- 目標：Chrome / Edge / Safari / Firefox 最新版；iOS Safari 觸控完整支援。
- 動畫尊重 `prefers-reduced-motion`。
